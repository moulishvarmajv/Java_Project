# 1. Introduction

This project is a Java-based Facial Recognition Attendance System developed using Object-Oriented Programming (OOP) principles and Maven. The primary objective is to automate attendance marking using facial recognition while maintaining a clean and modular software architecture.

# 2. Why Java?

Java was selected because:

- Platform Independent (Write Once, Run Anywhere)
- Strong Standard Library
- Easy to maintain
- Widely used in enterprise applications
- Excellent support for Maven and OpenCV

# 3. What is Maven?

Maven is a **Build Automation** and **Dependency Management** tool for Java projects.

Instead of manually compiling every Java file and downloading external libraries ourselves, Maven automates everything.

Example:

Without Maven:

```
javac Student.java
javac Course.java
javac AttendanceRecord.java
```

With Maven:

```
mvn compile
```

Maven compiles the complete project automatically.

---

# 6. What is pom.xml?

POM stands for **Project Object Model**.

The `pom.xml` file stores all project information.

Example:

- Project Name
- Version
- Dependencies
- Java Version
- Build Plugins

Every Maven project contains a `pom.xml`.

# 8. Why Package Name is `com.moulish`?

Java package naming follows reverse-domain naming.

Examples:

```
com.google
com.microsoft
org.apache
```

Since I don't own a company domain, I used:

```
com.moulish
```

This makes all class names globally unique.

Example:

Instead of

```
Student
```

Java recognizes

```
com.moulish.model.Student
```

# 34. Why Exceptions?

Instead of

```
return null;
```

we throw meaningful exceptions.

Example:

```
throw new StudentNotFoundException("Student Not Found");
```

Advantages:

- Easier Debugging
- Better Error Messages
- Cleaner Code

---

# 35. Why Private Variables?

Private variables provide Encapsulation.

Other classes cannot directly modify object data.

Access is provided through:

- Getters
- Setters

---

# 36. Why Getters and Setters?

They provide controlled access to object variables.

Advantages:

- Encapsulation
- Data Validation
- Better Security

---

# 37. Why final Keyword?

Example:

```
private final String rollNumber;
```

Roll Number should never change after object creation.

Hence it is declared final.

---

# 38. Why Override toString()?
